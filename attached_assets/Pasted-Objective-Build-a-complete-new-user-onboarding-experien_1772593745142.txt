Tasks
T001: Backend — recent community activity endpoint
Blocked By: []

Files: server/routes.ts, server/storage.ts

Details: Add GET /api/runs/community-activity (public, no auth required). Returns up to 6 group runs completed in the last 14 days, with participant count and host name — used as social proof on the Discover empty state.

Add getRecentCommunityActivity() in storage.ts:

SELECT r.id, r.title, r.location_name, r.activity_type,
       r.distance AS planned_distance, r.completed_at,
       u.name AS host_name,
       COUNT(DISTINCT rp.user_id) AS participant_count
FROM runs r
JOIN users u ON u.id = r.host_id
LEFT JOIN run_participants rp
  ON rp.run_id = r.id AND rp.status IN ('confirmed', 'attended')
WHERE r.is_completed = true
  AND r.completed_at >= NOW() - INTERVAL '14 days'
GROUP BY r.id, u.name
ORDER BY r.completed_at DESC
LIMIT 6
Register the route before GET /api/runs/:id in routes.ts so it isn't swallowed by the param route. Returns { runs: [...] }.

Acceptance: GET /api/runs/community-activity returns an array (empty array if no data, up to 6 items each with participant_count, host_name, location_name, activity_type, completed_at).

T003: Frontend — Welcome onboarding modal
Blocked By: []

Files: app/(tabs)/index.tsx

Details: A full-screen modal shown exactly once, the first time a logged-in user opens the app. Controlled by AsyncStorage key fara_has_onboarded.

State logic:

const [showOnboarding, setShowOnboarding] = useState(false);
const [onboardSlide, setOnboardSlide] = useState(0);
useEffect(() => {
  if (!user) return;
  AsyncStorage.getItem("fara_has_onboarded").then(v => {
    if (!v) setShowOnboarding(true);
  });
}, [user?.id]);
const finishOnboarding = () => {
  AsyncStorage.setItem("fara_has_onboarded", "true");
  setShowOnboarding(false);
  setOnboardSlide(0);
};
