3. Map Behavior
A. User Location

On app load, request geolocation permission.

If granted:

Center map on user location.

Zoom level: 13.

If denied:

Fallback to Houston, TX.

Zoom level: 12.

B. Dynamic Run Loading

When map loads → fetch runs within visible bounds.

When user pans or zooms → debounce 300ms → fetch runs in new bounds.

Use map bounds:

NorthEast lat/lng

SouthWest lat/lng

Only return runs inside bounding box.

C. Filtering Logic

When fetching runs, also apply:

paceMin <= user selected paceMax

paceMax >= user selected paceMin

distanceMin <= selected distanceMax

distanceMax >= selected distanceMin

style must match one of selected styles (if any selected)

Filtering must happen server-side for scalability.
